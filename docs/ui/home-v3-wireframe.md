# Home v3 Wireframe

## 目标

把首页从“入口广场”收成“学员任务中心”，让用户在 3 秒内回答三个问题：

1. 我今天先做什么
2. 为什么推荐这个
3. 做完之后去哪

## 信息结构

首页只保留四层：

1. 顶部状态
2. 今日任务
3. 下一步推荐
4. 其他入口

## 线框

```text
+--------------------------------------------------------+
| 顶部导航: 登录/账号 | streak | level                   |
+--------------------------------------------------------+
| 顶部状态卡                                             |
| Good morning · 任务中心                                |
| 今天优先说明                                           |
| 词汇掌握进度 + sync 状态                               |
| streak / XP / level                                    |
+--------------------------------------------------------+
| 今日任务                                               |
| [主任务卡]                                             |
|   标题: 今天先做什么                                   |
|   说明: 为什么先做                                     |
|   主 CTA: 只有这一个高权重按钮                         |
|   完成后去: 下一步                                     |
|                                                        |
| [次级任务 1] 复习 or 路径推进                          |
| [次级任务 2] 轻任务                                    |
+--------------------------------------------------------+
| 下一步推荐                                             |
| 如果未设角色: 只在这里出现角色选择                     |
| 如果已设角色:                                          |
|   当前角色                                             |
|   推荐场景                                             |
|   为什么推荐                                           |
|   完成后建议去哪                                       |
+--------------------------------------------------------+
| 其他入口                                               |
| 学习 | 场景 | 练习 | 词根 | 进度                       |
| 保留可达性，但全部降级成次级区                         |
+--------------------------------------------------------+
```

## 优先级原则

1. 主任务只有一个高权重 CTA，防止首页再次变成导航页
2. 复习和轻任务保留，但降成次级任务卡
3. 角色路径不再永久占首屏，只在首次设置或主动调整时出现
4. 学习 / 场景 / 练习 / 词根 / 进度全部保留，但统一下沉到“其他入口”

## 旧模块下沉说明

1. 原 hero 继续学习按钮被吸收到“今日任务”主卡
2. 统计卡不再占据首屏主体，改由顶部状态卡承接关键信息
3. trainer 入口从 learner 首屏移除
4. 角色选择区从永久模块改成“下一步推荐”中的设置入口
5. Daily tip 不再占主流程，当前版本直接去掉

## 与其他包的边界

1. 不改 `trainer.html`
2. 不改 Supabase schema
3. 不改 `learn.html`
4. 首页只消费现有 `storage.js` / `review-queue.js` / `role-config.js` 提供的数据
